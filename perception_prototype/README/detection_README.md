# detection.py 사용법

YOLO 세그멘테이션 모델로 물체(버스바, 볼트/너트 등)를 검출해서, 그 물체의
중심 좌표(픽셀 기준)를 계산하는 파일입니다.

## 지금 상태 (중요)

- **아직 학습된 모델(.pt 파일)이 없어서, 실제로 실행해서 검증한 적이 없습니다.**
  문법 오류만 확인된 상태예요.
- 이 컴퓨터는 인터넷이 안 돼서 사전학습 모델 다운로드도 안 되는 상태였습니다.
  → **학습된 모델 생기면 반드시 한 번 실행해서 결과가 정상인지 확인 필요.**

## 실행 방법

1. 학습된 모델 파일(`.pt`)을 준비합니다.
2. `detection.py` 맨 위에 있는 이 줄을 찾아서

   ```python
   MODEL_PATH = "yolov8n-seg.pt"  # TODO: 학습된 모델(.pt)로 교체
   ```

   `"yolov8n-seg.pt"` 부분을 실제 모델 파일 경로로 바꿉니다. 예:

   ```python
   MODEL_PATH = "C:/models/busbar_bolt_seg.pt"
   ```

3. 터미널에서 실행합니다.

   ```
   python detection.py
   ```

4. 정상이면 이런 식으로 출력됩니다 (숫자는 예시).

   ```
   detected 2 objects
   {'label': '버스바', 'confidence': 0.91, 'box': (400.0, 300.0, 600.0, 500.0), 'mask_center': (512.3, 401.7), 'camera_3d': (-0.1, 0.03, 0.5)}
   {'label': '볼트', 'confidence': 0.87, 'box': (...), 'mask_center': (...), 'camera_3d': (...)}
   ```

## 코드가 하는 일 (간단 설명)

1. `detect_objects(image, model_path, depth)` — 사진 한 장을 YOLO 모델에 넣어서, 검출된 물체마다
   - `label`: 뭔지 (버스바/볼트 등, 모델이 학습한 이름 그대로 나옴)
   - `confidence`: 얼마나 확신하는지 (0~1)
   - `box`: 네모 박스 좌표
   - `mask_center`: **물체의 실제 모양을 기준으로 계산한 중심 좌표**
     (박스 중심이 아님 — 버스바가 ㄹ자 모양이라 박스 중심을 쓰면 빈 공간이
     중심으로 잡힐 수 있어서, 실제 채워진 모양의 무게중심으로 계산함)
   - `camera_3d`: `mask_center` + `depth`를 핀홀 카메라 공식으로 변환한 **카메라 기준 3D 좌표**
     (로봇 베이스 기준으로 바꾸는 건 아직 미구현 — 아래 "임시값 목록"의 `world_to_base` 항목 참고)

2. `main()` — 이 파일을 직접 실행했을 때만 도는 테스트 코드. 지금은 테스트용으로
   배터리 모듈팩 사진을 넣게 되어 있는데, 실제 버스바/볼트 테스트하려면
   이 부분의 이미지 경로도 바꿔서 확인하면 됩니다.

## 지금 코드에 들어있는 임시값 목록 (실제 값 확인되면 교체 필요)

| 이름 | 지금 값 | 실제 값인가? | 설명 |
|---|---|---|---|
| `MODEL_PATH` | `"yolov8n-seg.pt"` | ❌ 가짜 | 학습된 모델이 아직 없어서 자리만 잡아둔 이름. 학습 끝나면 실제 `.pt` 경로로 교체 |
| `D455_HFOV_DEG`, `D455_VFOV_DEG` | `87.0`, `62.0` | ❌ 추정값 | 이 카메라(팔 끝 D455)의 실제 Isaac Sim 설정이 아니라, D455 카메라 스펙 문서 기준으로 추정 |
| `IMAGE_WIDTH`, `IMAGE_HEIGHT` | `1280`, `720` | ❌ 추정값 | 이 카메라가 실제로 몇 x 몇 해상도로 찍는지 확인 안 됨. D455 흔한 기본값으로 임시로 잡음 |
| `DUMMY_DEPTH_M` | `0.5` | ❌ 가짜 | 실제 depth 이미지가 없어서 "무조건 0.5m 떨어져있다" 치고 계산 공식만 검증한 값 |
| `CAMERA_TO_GRIPPER_OFFSET` | `translate=(0, 0.045, 0.05)`, `rpy=(-90, 0, -90)` | ✅ **실제 값** | Isaac Sim의 `Collected_m0609_camera/m0609_camera.usd`에서 직접 확인한 값 (2026-07-21). 이건 가짜 아님 |
| `world_to_base()` | 함수만 있고 계산 안 됨 (`NotImplementedError`) | — | 로봇 팔이 움직이는 동안 카메라 위치가 계속 바뀌어서, 그 순간의 관절 각도(FK 정보)가 있어야 계산 가능. 그 정보를 아직 못 받아옴 |

**요약**: `CAMERA_TO_GRIPPER_OFFSET` 빼고는 전부 실제 확인 안 된 임시값입니다. 실제 카메라 설정, 실제 depth 이미지, 실제 학습된 모델이 준비되면 이 표에 있는 값들부터 순서대로 교체하면 됩니다.
